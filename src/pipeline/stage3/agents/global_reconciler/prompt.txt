### TASK
1. **Identify Redundancies**: Scan for tables that represent the same logical entity but have different names (e.g., `EQUITY` vs `SYMBOL`).
   - Criteria for Merge: Identical Primary Keys, or 70%+ overlap in column meanings.
2. **Standardize Naming**: If different tables use different names for the same concept (e.g., `user_id` vs `account_id`), pick the most industry-standard one and rename them globally.
3. **Consolidate Dimensions**: Ensure dimension tables are clean and not fragmented.

### ACTION TAGS
- `MERGE_TABLES`: Move all columns and relationships from a source table to a target table.
- `RENAME_COLUMN`: Standardize cross-table column names.
- `DELETE_COLUMN`: Remove unnecessary duplicates.
- `ADD_RELATIONSHIP`: Define missing links.
